# 测试工程师文档（QA）

## 1. 角色目标

- 负责功能验证、回归测试与质量风险拦截。
- 覆盖核心链路：发送消息 -> Agent 执行 -> 结果回显 -> 日志留存。

## 2. 测试范围

- 团队页：群发/单发、状态流转、日志行内展示、快捷键行为。
- 配置页：编辑参数、保存、重启运行时后配置生效。
- 日志页：normal/error 筛选、CSV 导出文件完整性。
- 文件页：切换目录、树结构可展开。
- 终端页：命令执行、stdout/stderr 回显、按钮禁用恢复。

## 3. 关键用例

- 用例1：群发消息时 4 个 Agent 全部进入 RUNNING，完成后分别回到 DONE/FAILED。
- 用例2：单发给某 Agent，仅该 Agent 状态变化，其他 Agent 保持不变。
- 用例3：模拟 CLI 非 0 退出码，UI 显示 FAILED 且日志级别为 error。
- 用例4：路径相关提问时，回复中包含完整工作路径文本。
- 用例5：保存 `config/teams.yaml` 后重启应用，参数保持一致。

## 4. 回归基线

- `需求文档.md`
- `功能清单.md`
- `config/teams.yaml`

## 5. 缺陷提单要求

- 必填：页面、步骤、实际结果、期望结果、复现概率。
- 附件：日志截图或导出的 CSV 证据。
- 分级建议：
  - P0：崩溃/数据丢失/主链路不可用
  - P1：核心功能异常但有绕过方案
  - P2：体验或边缘场景问题
