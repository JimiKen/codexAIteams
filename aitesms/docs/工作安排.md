# 项目工作安排（PM）

## 当前问题与目标
- 团队列表执行日志可读性不足，需要支持更高显示和可拖拽调整。
- 团队成员需要“参与/休息”控制，可临时让成员停止参与任务。
- 日志页存在“看不到正常日志”的问题，需要修复筛选与落盘。
- 终端页输出不稳定，需要保证命令执行结果可见。

## 分工安排
1. 后端工程师（BE）
- 数据模型：维护 `enabled` 参与状态，支持持久化到 `config/teams.yaml`。
- 调度能力：派发任务时自动跳过“休息”成员；切休息时支持终止该成员当前任务。
- 日志能力：修复日志筛选逻辑，确保 UI 可见；运行日志追加写入 `logs/runtime.log`。
- 终端能力：改为 PowerShell 执行并实时输出，附带退出码。

2. 前端工程师（FE）
- 团队页布局：支持上下可拖拽（执行面板/团队对话）。
- 表格交互：列宽可拖拽、行高可拖拽；执行日志显示高度提升。
- 交互控件：新增“参与/休息”下拉控件并联动状态色。

3. 测试工程师（QA）
- 先停止参与当前迭代的执行工作（状态切为“休息/不参与”）。
- 暂不分配新增测试任务，待本轮修复完成后恢复。

## 本轮执行顺序
1. BE 完成模型与调度修复。
2. FE 完成团队页可视与交互修复。
3. BE/FE 联调日志与终端输出。
4. QA 恢复后执行回归测试。

## 验收标准
- 团队列表可把“测试工程师”切为“休息”，且不会再接收新任务。
- 执行日志在团队页可清晰查看，表格宽高可拖拽。
- 日志页可看到 normal/error 数据，并可导出 CSV。
- 终端页可稳定看到命令输出与退出码。
